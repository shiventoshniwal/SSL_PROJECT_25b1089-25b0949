\subsection{Gameplay}

\begin{itemize}[leftmargin=1.5em]
    \item \textbf{Welcome modal} on starting game with start button and game information (simple styling with basic properties like font-color,background-color,font style,size and alignment) and asks for username which if not entered leads to display of alert message(checking done after removing whitespaces in username). (\ref{fig:start})\\
    \item Storing \textbf{snake} as an array of pairs of coordinates.(To keep track of position of snake body. Per move new element is added at start of snake array and last element is popped out, and remaining elements are kept unchanged)\\
    \item Taking \textbf{keyboard input} is used to specify direction of motion of snake .(snake cannot reverse direction in one move).Head value gets updated (Keyboard inputs taken by event listeners)\\
    \item Storing \textbf{food items} as array of pairs of coordinates.(Foods  of have variables like name,value,color,immunity,speed).\\
    \item Every 200ms, the \textbf{boxes} corresponding to coordinates of snake and foods are \textbf{colored}.(So it appears that the snake is moving by seeing difference between succesive renderings)(\ref{fig:play})\\
    \item \textbf{Snake dies} when the coordinates of head either coincide with another element of snake array or when it leaves the grid.(Head coordinates outside grid boundaries)\\
    \item 5 different \textbf{types of fruits} that the snake can feed on.(Different properties of foods like immunity,length growth and speedup) One food item instantly appears when the previous is eaten.(Only one food at a time is displayed and it is selected randomly from food array. Generation position is also random).\\
    \item \textbf{Increasing length} by adding a copy of last "food value" number of elements of snake array to the array, so when snake moves after next "food value" moves we can see those elements(growth of snake) that would otherwise have been popped out).  \\
    \item Setting a 10s \textbf{immunity} timer in which the snake does not die on either hitting the wall or itself, if it has eaten a food that gives immunity.\\
    \item Setting a 10s \textbf{speed up} by reducing the time interval between successive renderings of the game(board,snake,food) temporarily, if snake eats a food that gives speed up.(So the gap in which click has to be done is reduced and gameplay becomes faster)\\
    \item Displays game-play \textbf{statistics} of current session(username,length of snake,duration of play,cause of death,session high score) on a \textbf{game-over modal} and updates session high score if needed.(\ref{fig:over})\\
\end{itemize}

\subsection{How layers communicate}
\begin{itemize}
\item At first, the user interacts with JavaScript front end framework. When the game ends, the front end sends the final score to Flask Server using fetch API. \\
\item Request is POST with a JSON body. Inside app.py, the score is parsed, validated and appended to history.txt with timestamp. \\
\item Managing history.txt and backup files is done by admin.sh. We have used bash commands like sed, awk, grep, cut, sort and case to create a menu driven interface with different options.\\
\end{itemize}
\subsection{Format for history.txt}

[timestamp] name \textbar~score  cause  duration \\ \\Chosen because with this format, sorting becomes easier and since game data is space separated, it can be handled by awk.

\subsection{Important functions defined in various files}
\subsubsection{ui.js}
\begin{itemize}
\item \textbf{init()}: For checking username\\
\item \textbf{showGameOver(score,cause,duration)}: For showing Game Over modal.\\
\end{itemize}
\subsubsection{game.js}
\begin{itemize}
\item \textbf{init()}:Initializing snake,food and calling handleInput() (for keyboard input),draw() (initial grid) and setting interval for loop() (200 ms).\\
\item \textbf{spawnFood()}:Defining food types and ensuring food generates where snake is not there.\\
\item \textbf{loop()}:Updating position of snake head and checking if snake is dead(and cause of death).Calling speedUp() and startImmunity() if needed. Increasing snake length if needed. Calling draw() for rendering.\\
\item \textbf{speedUp()}: Increasing snake speed for 10s by reducing the time between renderings.\\
\item \textbf{startImmunity()}: Setting immunity timer for 10s.\\
\item \textbf{draw()}: Drawing grid and displaying snake and fruits( by coloring corresponding positions).\\
\item \textbf{handleInput()}:Updates direction of motion of snake as per keyboard commands(Arrow keys or WASD).\\
\item \textbf{endGame(cause)}: Terminating running interval loops and calling showGameOver in ui.js, saveScore in api.js.\\

\end{itemize}
\subsubsection{api.js}
\begin{itemize}
    \item \textbf{saveScore(gameData)}:Sends score using fetch API\\
\end{itemize}
\subsubsection{app.py}
\begin{itemize}
    \item \textbf{index}: Serves frontend to the browser.\\
    
    \item \textbf{save\_score}: Validates received game data and appends it to history.txt in the decided format.\\
\end{itemize}


\section{Usage}
\begin{itemize}
\item Running game on server\\
\begin{enumerate}
    
\item Type 'python3 app.py' on command line.\\ 
\item Navigate to http://127.0.0.1:5000 on browser.\\
\item Enter username and press Start Game.\\ 
\item Can use arrow keys or WASD for snake motion.\\ 
\item Press Play Again to restart if game over

\end{enumerate}
\item Running admin.sh \\
\begin{enumerate}
\item Type 'bash admin.sh' on command line.\\ 
\item Navigate menu by entering number corresponding to required option.\\ \item Where valid, enter required suboptions out of those visible
\end{enumerate}
\end{itemize}
\section{Hurdles faced and how we overcame them}

\begin{itemize}[leftmargin=1.5em]
    \item Adjusting the display size of the grid and modal - By trial and error.\\
    \item Adjusting timers for changing speed of snake and setting immunity - Disabling existing loop and setting timer for running a new loop temporarily.\\
    \item Ensuring proper parsing and sorting of data in history.txt. For example, if username matches WALL or SELF - By using proper delimiters as per history.txt's format.\\
    \item Ensuring that a fruit does not generate on snake's body - Keeping in mind that the randomly generated fruit appears on the grid excluding all the coordinates inside snake array.\\
\end{itemize}
